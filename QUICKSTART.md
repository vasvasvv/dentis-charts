# Швидкий старт - Виправлення прив'язки лікарів до пацієнтів

## 🚀 За 5 хвилин

### 1. Резервна копія (ОБОВ'ЯЗКОВО!)
```bash
wrangler d1 backup create dentis-db
```

### 2. Міграція бази даних
```bash
wrangler d1 execute dentis-db --file=./migration.sql
```

### 3. Призначити лікарів існуючим пацієнтам
```bash
# Відкрийте Cloudflare Dashboard -> D1 -> Console і виконайте:
UPDATE patients 
SET doctor_id = (SELECT id FROM users WHERE role_id = 2 LIMIT 1);
```

### 4. Деплой Worker
```bash
npm run deploy
```

### 5. Перезавантажте сторінку в браузері
Готово! ✅

## ✅ Що було виправлено

- **Додано поле `doctor_id` в таблицю `patients`** - тепер пацієнти прив'язані до лікарів в БД
- **Виправлено `/api/doctors`** - повертає тільки користувачів з роллю "doctor"
- **Додано API для оновлення/видалення пацієнтів**
- **Додано методи в ClinicContext** - updatePatient, deletePatient, addVisit, deleteVisit
- **Покращено VisitModal** - тепер можна вибирати лікаря при створенні візиту
