## 🎯 Налаштування (одноразово)

### 1. Отримати Cloudflare credentials

```bash
# API Token
https://dash.cloudflare.com/profile/api-tokens
→ Create Token
→ "Edit Cloudflare Workers"
→ Create Token
→ СКОПІЮВАТИ!

# Account ID
https://dash.cloudflare.com/
→ Вибрати проект
→ Sidebar: "Account ID"
→ Скопіювати
```

### 2. Додати GitHub Secrets

```
Ваш репозиторій на GitHub
→ Settings
→ Secrets and variables
→ Actions
→ New repository secret

Додати:
1. CLOUDFLARE_API_TOKEN (токен з кроку 1)
2. CLOUDFLARE_ACCOUNT_ID (ID з кроку 1)
```

### 3. Закоммітити зміни

```bash
git add .github/
git commit -m "ci: налаштувати автоматичний деплой на Cloudflare"
git push
```

### 4. Створити Cloudflare Pages проект

**Опція A - Автоматично (рекомендовано):**

Просто зробіть push (крок 3), GitHub Actions автоматично створить проект.

**Опція B - Вручну:**

```
https://dash.cloudflare.com/
→ Pages
→ Create a project
→ Connect to Git
→ Вибрати репозиторій
→ Framework: Vite
→ Build command: npm run build
→ Output: dist
→ Save and Deploy
```

## ✨ Переваги автоматизації

### 1. Швидкість
- **Ручний деплой:** ~10-15 хвилин
- **Автодеплой:** ~2-3 хвилини (в фоні)

### 2. Надійність
- Завжди однаковий процес
- Неможливо пропустити крок
- Автоматичне тестування (якщо налаштовано)

### 3. Preview versions
- Кожен PR отримує свій preview URL
- Можна тестувати перед мерджем
- Автоматично видаляються після мерджу

### 4. Історія
- Всі деплої записані в GitHub
- Легко знайти коли і що деплоїлось
- Можна відкотитись до попередньої версії

### 5. CI/CD ready
- Легко додати тести
- Легко додати лінтинг
- Готово до розширення

## 🔧 Troubleshooting

### Workflow не запускається

```bash
# Перевірте що файли закоммічені:
git status
git add .github/
git commit -m "ci: add workflows"
git push
```

### Помилка authentication

```
GitHub → Settings → Secrets

Перевірте:
✅ CLOUDFLARE_API_TOKEN існує
✅ CLOUDFLARE_ACCOUNT_ID існує
✅ Токен має правильні права
```

### Помилка "Project not found"

```
Створіть проект в Cloudflare:
https://dash.cloudflare.com/ → Pages → Create project

АБО

Змініть назву в .github/workflows/cloudflare-pages.yml:
  projectName: ваша-назва
```

### Білд падає з помилкою

```bash
# Перевірте локально:
npm install
npm run build

# Якщо локально працює, перевірте логи в Actions tab
```

## 🎨 Додаткові можливості

### Додати тести перед деплоєм

У `.github/workflows/cloudflare-pages.yml` додайте перед Deploy:

```yaml
- name: Run tests
  run: npm test

- name: Run linting
  run: npm run lint
```

### Деплоїти з різних гілок

У `.github/workflows/cloudflare-pages.yml` змініть:

```yaml
on:
  push:
    branches:
      - main
      - develop      # ← Додайте гілки
      - staging
```

### Налаштувати environment variables

У workflow додайте:

```yaml
- name: Deploy
  uses: cloudflare/pages-action@v1
  with:
    ...
  env:
    NODE_ENV: production
    VITE_API_URL: ${{ secrets.API_URL }}
```
