# 🛠️ Виправлення помилки деплою на Cloudflare Pages

## 📌 Проблема

При деплої проекту на Cloudflare Pages виникала помилка:

```
✘ [ERROR] The detected framework ("Hono") cannot be automatically configured.
```

## 🔍 Причина

Ваш проект містить два компоненти:
1. **React frontend** (основний додаток) - має деплоїтися на Cloudflare Pages
2. **Hono API worker** (backend) - має деплоїтися на Cloudflare Workers

Cloudflare Pages виявив файли `src/worker.ts` та `wrangler.toml` і вирішив, що весь проект - це Hono worker, що призвело до конфлікту.

## ✅ Виправлення

Я створив наступні файли та зміни:

### 1. `.cfignore` - Ігнорування worker файлів
Цей файл каже Cloudflare Pages ігнорувати backend файли при білді frontend:
```
src/worker.ts
wrangler.toml
.wrangler/
```

### 2. `wrangler.toml` - Оновлена конфігурація
Видалив конфліктні секції [build] щоб уникнути плутанини з Cloudflare Pages.

### 3. `.cloudflare/pages.json` - Явна конфігурація
Створив явну конфігурацію для Cloudflare Pages із вказівкою використовувати Vite.

### 4. Оновлений `package.json`
Додав скрипт `deploy:worker` для окремого деплою API worker.

### 5. Документація
- `QUICK_FIX.md` - швидке виправлення (українською і англійською)
- `CLOUDFLARE_DEPLOY.md` - детальна інструкція з деплою
- `CHECKLIST.md` - покроковий чеклист
- `DEPLOY_FIX_SUMMARY.md` - технічний опис змін

## 🚀 Як задеплоїти зараз

### Варіант 1: Через Cloudflare Dashboard (НАЙШВИДШЕ)

1. Зайдіть у ваш проект на Cloudflare Pages
2. Перейдіть: **Settings** → **Builds & deployments**
3. Змініть налаштування:
   - **Framework preset:** `Vite` (НЕ "None", НЕ "Auto")
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
4. Збережіть та натисніть **Retry deployment**

### Варіант 2: Через Git

```bash
# 1. Закоммітити всі нові файли
git add .
git commit -m "fix: налаштувати Cloudflare Pages для деплою Vite проекту"
git push

# Cloudflare Pages автоматично запустить новий деплой
```

## 📋 Що робити далі

1. ✅ **Закоммітити зміни** (якщо ще не зробили):
   ```bash
   git add .
   git commit -m "fix: configure Cloudflare Pages deployment"
   git push
   ```

2. ✅ **Перевірити налаштування в Cloudflare Dashboard**
   - Framework preset = `Vite`
   - Build command = `npm run build`
   - Output = `dist`

3. ✅ **Запустити деплой**
   - Cloudflare автоматично запустить білд після push
   - АБО натисніть "Retry deployment" в dashboard

4. ⏳ **Опціонально: Задеплоїти API backend**
   Якщо потрібен backend:
   ```bash
   # Налаштувати D1 базу даних
   npx wrangler d1 create dentis-charts
   
   # Запустити міграції
   npx wrangler d1 execute dentis-charts --file=./schema.sql
   
   # Встановити JWT secret
   npx wrangler secret put JWT_SECRET
   
   # Задеплоїти worker
   npm run deploy:worker
   ```

## ⚠️ Якщо помилка все ще виникає

Спробуйте тимчасово видалити `wrangler.toml`:

```bash
git mv wrangler.toml wrangler.toml.backup
git add .
git commit -m "temp: видалити wrangler.toml для деплою Pages"
git push
```

Після успішного деплою frontend можна повернути файл назад для деплою worker.
